#基于SpringBoot框架的支付图谱
---
##项目背景
&#8195;&#8195;实际生活中大部分都是多关系、多维度的实体，传统关系数据库如Sql server、Mysql等虽然能够用结构化的方式统一存储管理实体数据，却难以表现其错综复杂的关系。

在支付场景下也是如此，账户的复杂交易与多级关联账户需要一个富有表现力的底层数据存储结构，图数据库则是一种恰当的选择。在支付方面，各账户之间存在不定的交易往来，

需要实现一个能在底层提供数据存储支撑，在前端清楚表示账户交易关联的项目。
##涉及技术
- SpringBoot（整体框架）


- Neo4j：热门图数据库 通过Spring-data-neo4j注解建立依赖


- d3.js：提供可视化支撑，良好实现点线的绘制

##运行环境
- Java8及以上


- 占用端口：7474、8081


- 数据库：neo4j社区版3.3.9

##功能介绍
- 查询账户

&#8195;&#8195;&#8195;输入账户的名称或者账号，web端显示该账户，点击列表显示详细信息


- 扩展账户关联点

&#8195;&#8195;&#8195;点击web端已有结点，浮框上出现的扩展按钮，点击展示出与该账户有关联的结点


- 筛选结点类型和关系类型

&#8195;&#8195;&#8195;点击筛选结点和筛选关系，出现下拉框，可以选择想要观察的结点类型和关系类型


##项目结构
- 控制器包

&#8195;&#8195;&#8195;控制器包里两个类，一个用于路由前端的通讯，接受参数调用相应的函数，另一个用于控制页面的链接对应资源

- 实体包

&#8195;&#8195;&#8195;实体包内定义了业务场景中对应的实体，两个父类Node和Relationship代表了两大类别，Node类有账户、商户等，Relationship有交易、账户绑定等

- 数据依赖包

&#8195;&#8195;&#8195;利用spring注解和已定义的实体类别，通过图数据库neo4j查询语句cypher实现数据的持久化依赖

- 功能函数包

&#8195;&#8195;&#8195;包含完成各个具体功能的函数

##注意事项
数据查询利用neo4j-bolt方式，项目使用时需要保持图数据库开启
